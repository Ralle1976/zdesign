   527→Agent: Full-Stack Developer - A11y Score Discrepancy Fix
   528→Task: Fix A11y score discrepancy between chat quality evaluation (100) and Props Panel badge (0)
   529→
   530→Work Log:
   531→
   532→### Root Cause Analysis
   533→
   534→The A11y score discrepancy had three root causes:
   535→
   536→1. **Client-side scoring too harsh** (`accessibility.ts`): `calculateAccessibilityScore` used flat penalties (critical=15, warning=7, info=2) that accumulated quickly on large design trees. A typical fallback design with 7+ contrast issues would score 0.
   537→
   538→2. **Server-side evaluation too lenient** (`evaluate/route.ts`): `evaluateAccessibility` only checked for hardcoded color patterns (`#94a3b8` on `#ffffff`) instead of actual WCAG contrast ratio calculation. It also missed nav aria-labels, section headings, and image alt text checks properly.
   539→
   540→3. **Fallback designs lacked accessibility metadata**: Navigation, footer, hero, and section elements had no `ariaLabel` or `role` attributes. Images had no `alt` text. This caused the client-side scanner to find many issues.
   541→
   542→4. **DesignRenderer ignored accessibility attributes**: Even when meta had `role`, `ariaLabel`, `alt`, the renderer didn't apply them to DOM elements.
   543→
   544→5. **Heading hierarchy detection was broken**: The scanner used `componentRef` to determine heading level but ignored the `tag` property (e.g., `tag: 'h1'`). Most fallback designs set heading level via `tag`, not `componentRef`.
   545→
   546→6. **Large text detection logic was wrong**: The original code used `&&` instead of `||` for WCAG large text criteria, making it nearly impossible for text to be classified as "large."
   547→
   548→### Fixes Applied
   549→
   550→#### 1. Fixed `calculateAccessibilityScore` in `/src/lib/accessibility.ts`
   551→- **Before**: Flat penalty accumulation — each critical = -15, warning = -7, info = -2. Score easily collapsed to 0 on large designs.
   552→- **After**: Per-category weighted scoring system. Each of 6 categories (contrast, alt-text, labels, headings, touch-target, semantics) is scored independently 0-100 with capped deductions, then averaged with weights (contrast=0.30, alt-text=0.20, labels=0.20, headings=0.10, touch-target=0.10, semantics=0.10). This produces realistic scores in the 50-80 range for fallback designs.
   553→
   554→#### 2. Fixed heading level detection in `/src/lib/accessibility.ts`
   555→- **Before**: Only checked `meta.componentRef` for heading level, ignoring `tag: 'h1'`.
   556→- **After**: Checks `tag` first (e.g., `h1` → level 1), then falls back to `componentRef`.
   557→
   558→#### 3. Fixed large text detection in `/src/lib/accessibility.ts`
   559→- **Before**: Buggy `&&` logic that required both conditions simultaneously.
   560→- **After**: Correct WCAG logic: large text = ≥24px OR (≥18px AND bold ≥700 weight).
   561→
   562→#### 4. Fixed image alt text check in `/src/lib/accessibility.ts`
   563→- **Before**: Checked `!node.content` which passed for images with URL content (URLs are not alt text).
